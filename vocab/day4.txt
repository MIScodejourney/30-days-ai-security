 Local Uploads
آپلود محلی — ذخیره فایل در نزدیک‌ترین دیتاسنتر به کاربر قبل از انتقال نهایی

 Object
آبجکت / فایل ذخیره‌شده در سیستم Object Storage

Bucket
باکت — محل نگهداری فایل‌ها در Object Storage

Bucket Region
منطقه اصلی ذخیره‌سازی باکت

 Replication
تکثیر — کپی کردن داده به محل دیگر

 Replication Task
وظیفه تکثیر — فرآیند انتقال فایل از مبدا به مقصد

 Cloudflare Queues
سیستم صف پیام برای مدیریت و زمان‌بندی پردازش تسک‌ها

 Queue
صف — محل انتظار برای پردازش وظایف

 Sharding
تقسیم‌بندی — پخش کردن بار بین چند صف یا سرور

 Dead Letter Queue (DLQ)
صف پیام‌های شکست‌خورده که چند بار Retry شده‌اند

 Metadata
فراداده — اطلاعات توصیفی درباره فایل (نام، اندازه، مسیر و ...)

Atomic Operation
عملیات اتمی — یا کامل انجام می‌شود یا اصلاً انجام نمی‌شود

Pending Replica
نسخه در انتظار ایجاد

Replica
نسخه کپی‌شده از داده

Replication Plan
برنامه کامل تکثیر (مبدا، مقصد، تعداد نسخه‌ها، اولویت و ...)

Replication Mode
نوع تکثیر (مثلاً فوری یا زمان‌بندی‌شده)

Priority
اولویت پردازش

Timestamp
مُهر زمانی — ثبت زمان دقیق

Replication Task Marker
نشانگر وظیفه تکثیر برای ارسال به صف

At-least-once Delivery
تضمین اینکه پیام حداقل یک بار پردازش شود

Enqueue
اضافه کردن یک پیام به صف

Pull Model
مدل کششی — مصرف‌کننده خودش تسک را از صف می‌گیرد

Polling
بررسی دوره‌ای صف برای دریافت تسک جدید

Batch
دسته‌بندی چند تسک برای پردازش همزمان

Durable Object
آبجکت پایدار برای نگهداری وضعیت و متادیتا

Garbage Collection
حذف داده‌های غیرضروری از سیستم

Dispatch
ارسال تسک به یک سرویس دیگر

Cross-regional Data Movement
انتقال داده بین مناطق جغرافیایی مختلف
